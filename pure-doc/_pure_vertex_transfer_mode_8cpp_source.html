30 https://developer.nvidia.com/gpugems/gpugems/part-v-performance-and-practicalities/chapter-28-graphics-pipeline-performance
31
32 GeForce 2 cards (and probably GeForce 4MX) only supported 16-bit indexing in HW, so
33 if you went over 64Ki vertices then it would have to do the indexing in software.
34
35 Some manual say that glDrawElements()' type parameter must be GL_UNSIGNED_BYTE or GL_UNSIGNED_SHORT, while some allow GL_UNSIGNED_INT too.
36 Depends on implementation. This also means that we must use the smallest type given to glDrawElements() not only because of
37 performance considerations but also in order to make the function work on older implementations.
38
39 glDrawRangeElements() indices and vertices limits:
40 http://www.opengl.org/registry/specs/EXT/draw_range_elements.txt
41 The specification of glDrawElements does not allow optimal performance for some OpenGL implementations, however.
42 In particular, it has no restrictions on the number of indices given, the number of unique vertices referenced nor a
43 direct indication of the set of unique vertices referenced by the given indices. This forces some OpenGL
44 implementations to walk the index data given, building up a separate list of unique vertex references for later use
45 in the pipeline. Additionally, since some OpenGL implementations have internal limitations on how many vertices they can
46 deal with simultaneously the unbounded nature of glDrawElements requires the implementation to be prepared to segment the
47 input data and do multiple passes. These preprocessing steps can consume a significant amount of time.
48 glDrawRangeElementsEXT is designed to avoid the preprocessing steps which may be necessary for glDrawElements.
49 As such it does not have the flexibility of glDrawElements but it is sufficiently functional for a large class of applications
50 to benefit from its use. glDrawRangeElementsEXT enhances glDrawElements in two ways:
51 1. The set of unique vertices referenced by the indices is explicitly indicated via the start and end parameters, removing the
52 necessity to determine this through examination of the index data. The implementation is given a contiguous chunk of vertex
53 data that it can immediately begin streaming through the vertex pipeline.
54 2. Recommended limits on the amount of data to be processed can be indicated by the implementation through GL_MAX_ELEMENTS_VERTICES_EXT
55 and GL_MAX_ELEMENTS_INDICES_EXT. If an application respects these limits it removes the need to split the incoming data into multiple
56 chunks since the maximums can be set to the optimal values for the implementation to handle in one pass.
57
58 http://www.opengl.org/discussion_boards/showthread.php/146440-glDrawRangeElements-GL_MAX_ELEMENTS_VERTICES-INDICES
59 However the performance effects of these caches has to do with the re-use of indices and
60 the relative locality of vertex array contents. These performance effects aren't really
61 related to the number of vertices actually processed by a particular glDrawRangeElements as
62 the limits above are specified to describe.
63 It's tempting to "re-interpret" these limits to be more "relevant" to how modern GPUs work, but
64 that is not what the limits are specified to mean.
65
66 http://stackoverflow.com/questions/7549991/questions-about-gldrawrangeelements
67 Now that we know how to use glDrawElements, let's talk about glDrawRangeElements.
68 When normally using glDrawElements, you specify a location in the source index array to pull from.
69 The indices and count parameters tell OpenGL where to find the indices. But the actual indices pulled from
70 this array could be anywhere within the boundaries of the source vertex array indices.
71 glDrawRangeElements allows you to give OpenGL a range (half-open again) of vertex index values.
72 What you are saying is that the indices it gets during this draw call will not exceed that range (half-open).
73 This could allow the driver to perform useful optimizations. The start value should be the lowest index value
74 that will be gotten from the index array, and the end should be the highest (+1 because it's half-open).
75 It should not simply be the index of the first and last vertices.
76
77 http://www.songho.ca/opengl/gl_vertexarray.html
78 The additional parameters in glDrawRangeElements() are start and end index, then OpenGL prefetches a limited
79 amount of vertices from these values: end - start + 1. And the values in index array must lie in between
80 start and end index. Note that not all vertices in range (start, end) must be referenced. But, if you specify a
81 sparsely used range, it causes unnecessary process for many unused vertices in that range.
82
83 ***************************
84
85 Regular VAs, compiled VAs, server-side VAs.
86
87 Compiled vertex array: lock-unlock, but only 1 can be locked at a time, so the only use of this may be
88 when the same geometry is rendered multiple times like multipass blending if no multitexturing available.
89 Some performance gain may be observed also when using indices (drawElements) because shared vertices will be
90 transformed and lit only once. This also means there is no such post-TnL cache optimisation with standard drawElements.
91
92 Early vendor-specific "VBO"s:
93 - GeForce2: VAR: NV_vertex_array_range, NV_vertex_array_range2
94 VRAM or AGP memory. Write is slow. Write is fast only if Fast Writes is supported.
95 Rule: allocate static geometry in VRAM, allocate AGP mem for dynamic geometry.
96 Modifying AGP memory is slow, so first work in system memory, then block-copy to AGP memory.
97 - VAO: ATI_vertex_array_object, ATI_map_object_buffer, ATI_element_array (like ELEMENT_ARRAY_BUFFER_ARB target).
98 Can't be AGP mem, only VRAM.
99
100 Usage params:
101 - ATI_vertex_array_object: STATIC_ATI, DYNAMIC_ATI
102 - NV_vertex_array_range: readFreq, writeFreq, priority
103 - ARB_vertex_buffer_object: STATIC, STREAM, DYNAMIC; DRAW, READ, COPY
104
105
106 BufferObject: a GPU allocated memory buffer
107 Vertex Buffer Object: a BufferObject containing vertices informations (colors, position, custom data used by a shader, ...)
108 Pixel Buffer Object: a BufferObject containing pixel or texel informations. Mainly used to upload textures.
109 Element Buffer Object: a BufferObject containing indices (used by glDrawElements).
110 Vertex Array: memory used by gl*Pointer call.
111 Might be host memory or a Vertex Buffer Object if it is bound using glBindBuffer command with GL_ARRAY_BUFFER.
112 Element Array: memory used by glDrawElements call.
113 Might be host memory or an Element Buffer Object if it is bound using glBindBuffer command with GL_ELEMENT_ARRAY_BUFFER.

196TPureBool PureVertexTransfer::PureVertexTransferImpl::setVertexTransferMode(TPURE_VERTEX_TRANSFER_MODE vtrans)
197{
198 _pOwner->getManagedConsole().OLnOI("PureVertexTransfer::setVertexTransferMode(%u)", vtrans);
199
200 if ( _pOwner->isLevel2() )
201 {
202 if ( !_pOwner->getManager() )
203 {
204 // being level-2 without a manager is unusual, but can be achieved when:
205 // either user invokes Detach() for a submesh of a complete level-1 mesh,
206 // or when we are still creating a mesh and it is not yet attached or it does not have any children.
207 // any case, we should IGNORE execution of this function.
208 _pOwner->getManagedConsole().EOLnOO("setVertexTransferMode(%u) of submesh called WITHOUT existing level-1 parent mesh, rejecting!", vtrans);
209 return false;
210 }
211
212 // submesh must reject if its setVertexTransferMode() was not called by its parent level-1 mesh but someone else from outside ...
213 // note: our manager in this case is a level-1 VertexTransfer (Mesh3D) mesh
214 if ( !((PureVertexTransfer*)_pOwner->getManager())->pImpl->bParentInitiatedOperation )
215 {
216 _pOwner->getManagedConsole().EOLnOO("setVertexTransferMode(%u) of submesh called outside of its level-1 parent mesh, rejecting!", vtrans);
217 return false;
218 }
219 }
220
221 // Note: we don't check if the selected transfer mode is already selected.
222 // Reason: selecting an already selected transfer mode is a way to recreate buffers / refresh geometry on server side.
223 // Modify geometry in system RAM and simply issue setVertexTransferMode( getVertexTransferMode() ) to apply changes.
224
225 if ( PureVertexTransfer::isVertexReferencingIndexed(vertexTransferMode) &&
226 !PureVertexTransfer::isVertexReferencingIndexed(vtrans) )
227 {
228 if ( !isSwitchFromIndexedAllowed() )
229 {
230 _pOwner->getManagedConsole().EOLnOO("ERROR: failed to switch from indexed to direct referencing mode!");
231 return false;
232 }
233 }
234
235 if ( !PureVertexTransfer::isVertexTransferModeSelectable(vtrans) )
236 {
237 _pOwner->getManagedConsole().EOLnOO("ERROR: given mode %u is unavailable!", vtrans);
238 return false;
239 }
240
241 if ( _pOwner->isLevel2() )
242 {
243 FreeGLresources(); // now it's safe to delete GL resources
244 vertexTransferMode = 0;
245
246 if ( BIT_READ(vtrans, PURE_VT_SVA_BIT) )
247 {
248 /* we want VBO */
249 if ( !compileIntoVertexBufferObjects(
250 PureVertexTransfer::isVertexReferencingIndexed(vtrans),
251 PureVertexTransfer::isVertexModifyingDynamic(vtrans) ) )
252 {
253 FreeGLresources(); // we cannot know exactly what was allocated and what no, make sure we don't keep anything!
254 _pOwner->getManagedConsole().EOLnOO("ERROR: compileIntoVertexBufferObjects() failed!");
255 return false;
256 }
257 }
258 else
259 {
260 /* check if we want display list? */
261 if ( (vtrans == (PURE_VMOD_STATIC | PURE_VREF_DIRECT)) ||
262 (vtrans == (PURE_VMOD_STATIC | PURE_VREF_INDEXED)) )
263 {
264 if ( !compileIntoDisplayList(PureVertexTransfer::isVertexReferencingIndexed(vtrans)) )
265 {
266 FreeGLresources(); // we cannot know exactly what was allocated and what no, make sure we don't keep anything!
267 _pOwner->getManagedConsole().EOLnOO("ERROR: compileIntoDisplayList() failed!");
268 return false;
269 }
270 }
271 }
272
273 /* no other transfer mode needs further preparation */
274 vertexTransferMode = vtrans;
275 }
276 else
277 {
278 bParentInitiatedOperation = true;
279 _pOwner->getManagedConsole().OLn("Ok, applying for subobjects ...");
280 for (TPureInt i = 0; i < _pOwner->getCount(); i++)
281 {
282 if ( !((PureVertexTransfer*)(_pOwner->getAttachedAt)(i))->setVertexTransferMode( vtrans ) )
283 {
284 _pOwner->getManagedConsole().EOLn("Failed a setVertexTransferMode(%u) for subobject %d!", vtrans, i);
285 FreeGLresources(); // we cannot know exactly what was allocated and what no, make sure we don't keep anything!
286 vertexTransferMode = 0;
287 bParentInitiatedOperation = false;
288 _pOwner->getManagedConsole().OO();
289 return false;
290 }
291 }
292 _pOwner->getManagedConsole().OLn("Done!");
293 bParentInitiatedOperation = false;
294 vertexTransferMode = vtrans;
295 }
296
297 _pOwner->getManagedConsole().OO();
298 return true;
299} // setVertexTransferMode()

369TPureUInt PureVertexTransfer::PureVertexTransferImpl::transferVertices()
370{
371 assert(_pOwner->isLevel2());
372 if ( !_pOwner->isLevel2() )
373 {
374 _pOwner->getManagedConsole().EOLn("ERROR: PureVertexTransfer::%s() invoked for level-1!", __FUNCTION__);
375 return 0;
376 }
377
378 /* currently not supporting any vendor-specific mode */
379 if ( BITF_READ(getVertexTransferMode(),PURE_VT_VENDOR_BITS,3) != 0 )
380 {
381 _pOwner->getManagedConsole().EOLn("ERROR: PureVertexTransfer::%s() unsupported vendor-specific mode!", __FUNCTION__);
382 return 0;
383 }
384
385 nLastTransferredVertices = _pOwner->getVertexIndicesCount();
386 nLastTransferredTriangles = _pOwner->getTriangleCount();
387 // increasing the number in the parent too, however it must be parent's logic to zero it out before iterating over its children again
388 ((PureVertexTransfer*)(_pOwner->getManager()))->pImpl->nLastTransferredVertices += nLastTransferredVertices;
389 ((PureVertexTransfer*)(_pOwner->getManager()))->pImpl->nLastTransferredTriangles += nLastTransferredTriangles;
390
391 // this function doesn't check for GL errors since this is also part of rendering,
392 // renderer should check for errors after each frame
393
394 if ( BIT_READ(getVertexTransferMode(), PURE_VT_VA_BIT) == 0u )
395 {
396 /* there is no array for geometry, so we either invoke a display list or pass vertices 1-by-1 */
397
398 if ( PureVertexTransfer::isVertexModifyingDynamic( getVertexTransferMode() ) )
399 {
400 // PURE_VT_DYN_..._1_BY_1
401 ProcessGeometry( PureVertexTransfer::isVertexReferencingIndexed( getVertexTransferMode() ) );
402 }
403 else
404 {
405 // PURE_VT_STA_..._DL
406 assert(nDispList > 0);
407 glCallList( nDispList );
408 }
409
410 return nLastTransferredVertices;
411 } // PURE_VT_VA_BIT == 0u
412
413 /* vertex array either in client or server */
414 const bool bServer = BIT_READ(getVertexTransferMode(), PURE_VT_SVA_BIT) == 1u;
415
416 SetArrayPointers( bServer );
417
418 // 1 big benefit of using CVA is when we want to do multitexturing with only
419 // 1 TMU, meaning that we need to do it in at least 2 passes, specifying
420 // vertex geometry twice! However even TNT had 2 TMUs, so I'm not planning to do this improvement.
421 // Another big benefit is when we have shared vertices, in indexed rendering
422 // all vertices are transformed if lock is enabled, thus shared vertices
423 // can be used multiple times EVEN THOUGH THEY ARE TRANSFORMED ONLY ONCE!
424 // Too bad we currently create redundant vertex geometry when loading OBJ mesh,
425 // however we have backlog item for that ...
426 if ( BIT_READ(getVertexTransferMode(), PURE_VT_CVA_BIT) == 1u )
427 glLockArraysEXT(0, _pOwner->getVerticesCount());
428
429 // Static and dynamic vmods are not differentiated here but when creating the arrays.
430 // That is why we don't check for vmod in this code.
431
432 if ( PureVertexTransfer::isVertexReferencingIndexed( getVertexTransferMode() ) )
433 {
434 assert(_pOwner->getVertexIndicesCount() != 0);
435 if ( BIT_READ(getVertexTransferMode(), PURE_VT_RNG_BIT) )
436 {
437 // TODO: should call this in loop, do multiple batches based on the implementation-dependent values
438 // (GL_MAX_ELEMENTS_VERTICES_EXT, GL_MAX_ELEMENTS_INDICES_EXT).
439 glDrawRangeElementsEXT(
440 getGLprimitiveFromPureprimitive(_pOwner->getPrimitiveFormat()),
441 _pOwner->getMinVertexIndex(false), _pOwner->getMaxVertexIndex(false),
442 _pOwner->getVertexIndicesCount(), _pOwner->getVertexIndicesType(false), bServer ? NULL : _pOwner->getVertexIndices(false) );
443 }
444 else
445 {
446 glDrawElements(
447 getGLprimitiveFromPureprimitive(_pOwner->getPrimitiveFormat()),
448 _pOwner->getVertexIndicesCount(), _pOwner->getVertexIndicesType(false), bServer ? NULL : _pOwner->getVertexIndices(false) );
449 }
450 }
451 else
452 {
453 assert(_pOwner->getVerticesCount() != 0);
454 glDrawArrays( getGLprimitiveFromPureprimitive(_pOwner->getPrimitiveFormat()), 0, _pOwner->getVerticesCount() );
455 }
456
457 if ( BIT_READ(getVertexTransferMode(), PURE_VT_CVA_BIT) == 1u )
458 glUnlockArraysEXT();
459
460 ResetArrayPointers( bServer );
461
462 return nLastTransferredVertices;
463}

517void PureVertexTransfer::PureVertexTransferImpl::ProcessGeometry(TPureBool indexed) const
518{
519 assert(_pOwner->isLevel2());
520
521 /* Make sure we don't call GL functions when no accelerator is present */
522 if ( !PureHwInfo::get().getVideo().isAcceleratorDetected() )
523 return;
524
525 //const TRGBAFLOAT* const pColors = _pOwner->getMaterial(false).getColors(0);
526 //const void* const pColorIndices = _pOwner->getMaterial(false).getColorIndices(0);
527 const TUVW* const pTexcoords = _pOwner->getMaterial(false).getTexcoords(0);
528 // following can be NULL if material system supports only 1 level
529 const TUVW* const pTexcoords2 = _pOwner->getMaterial(false).getTexcoords(1);
530 //const void* const pTexcoordIndices2 = _pOwner->getMaterial(false).getTexcoordIndices(1);
531
532 glBegin( getGLprimitiveFromPureprimitive(_pOwner->getPrimitiveFormat()) );
533 for (TPureUInt i = 0; i < _pOwner->getVertexIndicesCount(); i++)
534 {
535 const TXYZ& vertex = indexed ? _pOwner->getVertices(false)[ _pOwner->getVertexIndex(i) ] : _pOwner->getVertices(false)[i];
536 const TXYZ& normal = indexed ? _pOwner->getNormals(false)[ _pOwner->getVertexIndex(i) ] : _pOwner->getNormals(false)[i];
537 //const TRGBAFLOAT& color = indexed ? pColors[ _pOwner->getVertexIndex(pColorIndices, i) ] : pColors[i];
538 const TUVW& uvw = indexed ? pTexcoords[ _pOwner->getVertexIndex(i) ] : pTexcoords[i];
539
540 if ( PureHwInfo::get().getVideo().isMultiTexturingSupported() )
541 {
542 if ( _pOwner->getMaterial(false).isMultiTextured() )
543 {
544 assert(pTexcoords2 != NULL);
545 const TUVW& uvw2 = indexed ? pTexcoords2[ _pOwner->getVertexIndex(i) ] : pTexcoords2[i];
546 glMultiTexCoord2fARB(GL_TEXTURE0_ARB, uvw.u, uvw.v);
547 glMultiTexCoord2fARB(GL_TEXTURE1_ARB, uvw2.u, uvw2.v);
548 }
549 else
550 {
551 if ( _pOwner->getMaterial(false).getTexture() != PGENULL )
552 glMultiTexCoord2fARB(GL_TEXTURE0_ARB, uvw.u, uvw.v);
553 }
554 }
555 else
556 {
557 if ( _pOwner->getMaterial(false).getTexture() != PGENULL )
558 glTexCoord2f(uvw.u, uvw.v);
559 }
560
561 // per-vertex color is disabled for compatibility with tmcsgfxlib wrapper, reason somewhere in setarraypointers()
562 //glColor4f(color.red, color.green, color.blue, color.alpha);
563
564 // in the past, we sent normal data only if bAffectedLights was true
565 // however, probably it should be solved in a different way, like enable/disable specific lights like LIGHT0 before rendering object,
566 // anyway I've commented out this condition since after refactor, VertexTransfer does not know such thing like bAffectedByLights.
567 //if ( bAffectedByLights )
568 glNormal3f(normal.x, normal.y, normal.z);
569 glVertex3f(vertex.x, vertex.y, vertex.z);
570 } // for k
571 glEnd();
572} // ProcessGeometry()

626TPureBool PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects(TPureBool indexed, TPureBool dynamic)
627{
628 assert(_pOwner->isLevel2());
629
630 /* Make sure we don't call GL functions when no accelerator is present */
631 if ( !PureHwInfo::get().getVideo().isAcceleratorDetected() )
632 return false;
633
634 if ( nVerticesVBO != 0 )
635 {
636 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() already has vertex VBO!");
637 return false;
638 }
639 if ( _pOwner->getVertices(false) == NULL )
640 {
641 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() getVertices() is NULL!");
642 return false;
643 }
644 if ( _pOwner->getNormals(false) == NULL )
645 {
646 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() getNormals() is NULL!");
647 return false;
648 }
649
650 if ( indexed )
651 {
652 if ( nIndicesVBO != 0 )
653 {
654 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() nIndicesVBO != 0!");
655 return false;
656 }
657 if ( _pOwner->getVertexIndices(false) == NULL )
658 {
659 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() getVertexIndices() is NULL!");
660 return false;
661 }
662 if ( ( !pglGenBuffersARB(1, &nIndicesVBO) ) ||
663 ( !pglBindBufferARB(GL_ELEMENT_ARRAY_BUFFER_ARB, nIndicesVBO) ) ||
664 /* Note: we always store indices in static buffer but could be in client memory, too. */
665 ( !pglBufferDataARB(GL_ELEMENT_ARRAY_BUFFER_ARB, _pOwner->getVertexIndicesCount() * PureGLsnippets::getSizeofIndexType(_pOwner->getVertexIndicesType()), _pOwner->getVertexIndices(false), GL_STATIC_DRAW_ARB) ) ||
666 ( !pglBindBufferARB(GL_ELEMENT_ARRAY_BUFFER_ARB, 0) ) )
667 {
668 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create element array buffer!");
669 return false;
670 }
671 }
672
673 const GLenum usage = dynamic ? GL_DYNAMIC_DRAW_ARB : GL_STATIC_DRAW_ARB;
674
675 if ( ( !pglGenBuffersARB(1, &nVerticesVBO) ) ||
676 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, nVerticesVBO) ) ||
677 ( !pglBufferDataARB(GL_ARRAY_BUFFER_ARB, _pOwner->getVerticesCount() * sizeof(TXYZ), _pOwner->getVertices(false), usage) ) )
678 {
679 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create vertex VBO!");
680 return false;
681 }
682
683 assert(_pOwner->getMaterial(false).getColors() != NULL);
684 if ( ( !pglGenBuffersARB(1, &nColorsVBO) ) ||
685 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, nColorsVBO) ) ||
686 ( !pglBufferDataARB(GL_ARRAY_BUFFER_ARB, _pOwner->getMaterial(false).getColorsCount() * sizeof(TRGBAFLOAT), _pOwner->getMaterial(false).getColors(), usage) ) )
687 {
688 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create color VBO!");
689 return false;
690 }
691
692 assert(_pOwner->getMaterial(false).getTexcoords() != NULL);
693 if ( ( !pglGenBuffersARB(1, &nTexcoordsVBO[0]) ) ||
694 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, nTexcoordsVBO[0]) ) ||
695 ( !pglBufferDataARB(GL_ARRAY_BUFFER_ARB, _pOwner->getMaterial(false).getTexcoordsCount() * sizeof(TUVW), _pOwner->getMaterial(false).getTexcoords(), usage) ) )
696 {
697 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create texcoords VBO!");
698 return false;
699 }
700
701 if ( PureHwInfo::get().getVideo().isMultiTexturingSupported() )
702 {
703 if ( _pOwner->getMaterial(false).isMultiTextured() )
704 {
705 assert(_pOwner->getMaterial(false).getTexcoords(1) != NULL);
706 if ( ( !pglGenBuffersARB(1, &nTexcoordsVBO[1]) ) ||
707 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, nTexcoordsVBO[1]) ) ||
708 ( !pglBufferDataARB(GL_ARRAY_BUFFER_ARB, _pOwner->getMaterial(false).getTexcoordsCount(1) * sizeof(TUVW), _pOwner->getMaterial(false).getTexcoords(1), usage) ) )
709 {
710 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create 2nd texcoords VBO!");
711 return false;
712 }
713 }
714 }
715
716 if ( ( !pglGenBuffersARB(1, &nNormalsVBO) ) ||
717 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, nNormalsVBO) ) ||
718 ( !pglBufferDataARB(GL_ARRAY_BUFFER_ARB, _pOwner->getVerticesCount() * sizeof(TXYZ), _pOwner->getNormals(false), usage) ) ||
719 ( !pglBindBufferARB(GL_ARRAY_BUFFER_ARB, 0) ) )
720 {
721 _pOwner->getManagedConsole().EOLnOO("ERROR: PureVertexTransfer::PureVertexTransferImpl::compileIntoVertexBufferObjects() failed to create normals VBO!");
722 return false;
723 }
724
725 return true;

785void PureVertexTransfer::PureVertexTransferImpl::SetArrayPointers(TPureBool redirectToServer) const
786{
787 /* Make sure we don't call GL functions when no accelerator is present */
788 if ( !PureHwInfo::get().getVideo().isAcceleratorDetected() )
789 return;
790
791 // since this is called every render frame, I dont check for values with if, instead I use only assertions!
792
793 if ( (PureVertexTransfer::isVertexReferencingIndexed(vertexTransferMode)) && redirectToServer )
794 {
795 assert(nIndicesVBO != 0);
796 glBindBufferARB(GL_ELEMENT_ARRAY_BUFFER_ARB, nIndicesVBO);
797 }
798
799 glEnableClientState(GL_VERTEX_ARRAY);
800 // per-vertex color is disabled for compatibility with tmcsgfxlib wrapper, reason:
801 // as tmcsgfxlib uses a single vertex color for object color key when object is textured,
802 // we shouldn't specify any further per-vertex color, otherwise alphablending would be bad with legacy projects.
803 // In future, we should rather use glBlendColor() for such objects and then per-vertex colors could be re-enabled again.
804 // glEnableClientState(GL_COLOR_ARRAY);
805 glEnableClientState(GL_NORMAL_ARRAY);
806 if ( redirectToServer )
807 {
808 assert(nVerticesVBO != 0);
809 glBindBufferARB(GL_ARRAY_BUFFER_ARB, nVerticesVBO);
810 glVertexPointer(3, GL_FLOAT, 0, NULL);
811
812 // per-vertex color is disabled for compatibility with tmcsgfxlib wrapper, reason somewhere above
813 //glBindBufferARB(GL_ARRAY_BUFFER_ARB, nColorsVBO);
814 //glColorPointer(4, GL_FLOAT, 0, NULL);
815
816 assert(nNormalsVBO != 0);
817 glBindBufferARB(GL_ARRAY_BUFFER_ARB, nNormalsVBO);
818 glNormalPointer(GL_FLOAT, 0, NULL);
819 }
820 else
821 {

823 glVertexPointer(3, GL_FLOAT, 0, _pOwner->getVertices(false));

825 //glColorPointer(4, GL_FLOAT, 0, _pOwner->getMaterial(false).getColors());
826 assert(_pOwner->getNormals(false) != NULL);

850 }

996TPURE_VERTEX_TRANSFER_MODE PureVertexTransfer::selectVertexTransferMode(
997 TPURE_VERTEX_MODIFYING_HABIT vmod,
998 TPURE_VERTEX_REFERENCING_MODE vref,
999 TPureBool bForceUseClientMemory)
1000{
1001 TPURE_VERTEX_TRANSFER_MODE ret = 0;
1002
1003 if ( vref == PURE_VREF_INDEXED )
1004 ret = ret | PURE_VREF_INDEXED;
1005
1006 // dont go further if CPU is rendering
1007 if ( !PureHwInfo::get().getVideo().isAcceleratorDetected() )
1008 return ret | PURE_VMOD_DYNAMIC;
1009
1010 if ( vmod == PURE_VMOD_DYNAMIC )
1011 ret = ret | PURE_VMOD_DYNAMIC;
1012
1013 if ( PureHwInfo::get().getVideo().isDrawRangeElementsSupported() )
1014 ret = ret | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_RNG_BIT);
1015
1016 if ( PureHwInfo::get().getVideo().isCompiledVertexArraySupported() )
1017 ret = ret | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_CVA_BIT);
1018
1019 // bForceUseClientMemory is considered only if modifying habit is dynamic
1020 if ( bForceUseClientMemory && (vmod == PURE_VMOD_DYNAMIC) )
1021 return ret | BIT(PURE_VT_VA_BIT); // client-side should use an array, even if none of RNG and CVA bits are available, to avoid selecting immediate mode
1022
1023 // At this point we want to use host/server memory, but it could be still either client- or server-side array, or display list or immediate mode, so
1024 // we should set PURE_VT_VA_BIT only in case of some arraying.
1025
1026 // if general VBO is supported, return with that immediately as we don't want to stick to any vendor-specific magical stuff.
1027 if ( PureHwInfo::get().getVideo().isVertexBufferObjectSupported() )
1028 return ret | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_SVA_BIT);
1029
1030 if ( PureHwInfo::get().getVideo().isATIVertexArrayObjectSupported() )
1031 {
1032 if ( PureHwInfo::get().getVideo().isATIElementArraySupported() )
1033 ret = ret | BIT(PURE_VT_RNG_BIT) | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_SVA_BIT) | BITF_PREP(PURE_VT_VENDOR_ATI_2,PURE_VT_VENDOR_BITS,3);
1034 else
1035 ret = ret | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_SVA_BIT) | BITF_PREP(PURE_VT_VENDOR_ATI_1,PURE_VT_VENDOR_BITS,3);
1036 }
1037 else
1038 {
1039 if ( PureHwInfo::get().getVideo().isNVVertexArrayRangeSupported() )
1040 {
1041 if ( PureHwInfo::get().getVideo().isNVElementArraySupported() )
1042 ret = ret | BIT(PURE_VT_RNG_BIT) | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_SVA_BIT) | BITF_PREP(PURE_VT_VENDOR_NVIDIA_2,PURE_VT_VENDOR_BITS,3);
1043 else
1044 ret = ret | BIT(PURE_VT_VA_BIT) | BIT(PURE_VT_SVA_BIT) | BITF_PREP(PURE_VT_VENDOR_NVIDIA_1,PURE_VT_VENDOR_BITS,3);
1045 }
1046 }
1047
1048 // at this point, if VA_BIT is not set, the selected mode is either display list or immediate mode, depending on vmod
1049
1050 return ret;
1051} // selectVertexTransferMode()

1215TPureUInt PureVertexTransfer::getUsedVideoMemory() const
1216{
1217 // level-2 objects' vertex transfer mode might or might not be set, but anyway, always the transfer mode of their owner level-1 object counts!
1218 const TPURE_VERTEX_TRANSFER_MODE& transferMode =
1219 isLevel1() ?
1220 getVertexTransferMode() :
1221 ((PureVertexTransfer*)getManager())->getVertexTransferMode();
1222
1223 if ( !PureVertexTransfer::isVideoMemoryUsed(transferMode) )
1224 return 0;
1225
1226 TPureUInt sumSubMeshMemoryUsage = 0;
1227
1228 if ( isLevel1() )
1229 {
1230 for (TPureInt i = 0; i < getSize(); i++)
1231 {
1232 const PureVertexTransfer* const subobj = (PureVertexTransfer*) getAttachedAt(i);
1233 if ( subobj == PGENULL )
1234 continue;
1235 sumSubMeshMemoryUsage += subobj->getUsedVideoMemory();
1236 }
1237 return sumSubMeshMemoryUsage;
1238 }
1239
1240 // at this point this is a level-2, we keep using sumSubMeshMemoryUsage for its mem usage
1241
1242 // VBO or display list, it doesn't matter, we always calculate with vertices, normals, colors, and 1 or 2 texture uvw arrays,
1243 // since these are together always specified for both VBO and display lists.
1244 sumSubMeshMemoryUsage = getVerticesCount() * sizeof(TXYZ) * 2 /* *2 because of we have normals too */ +
1245 getMaterial(false).getColorsCount() * sizeof(TRGBAFLOAT) +
1246 getMaterial(false).getTexcoordsCount() * sizeof(TUVW);
1247
1248 if ( getMaterial(false).getTexcoords(1) != PGENULL )
1249 {
1250 sumSubMeshMemoryUsage += getMaterial(false).getTexcoordsCount(1) * sizeof(TUVW);
1251 }
1252
1253 // In case of VBO, we also add index (element) array if it is indexed rendering
1254 if ( (BIT_READ(transferMode, PURE_VT_SVA_BIT) == 1u) )
1255 {
1256 if ( pImpl->nIndicesVBO != 0 )
1257 {
1258 sumSubMeshMemoryUsage += getVertexIndicesCount() * PureGLsnippets::getSizeofIndexType(getVertexIndicesType());
1259 }
1260 }
1261
1262 return sumSubMeshMemoryUsage;
1263} // getUsedVideoMemory()
