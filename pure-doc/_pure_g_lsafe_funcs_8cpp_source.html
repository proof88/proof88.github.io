16// Note that since GL 4.3 there is callback mechanism that can be set up with glDebugMessageCallback().
17// We don't use it that though, mostly because our implementation is not at GL 4.3 level, and it would
18// need debug context, and I'm not sure how much performance penalty it makes.
19
20// The following functions below could be replaced by a single functions with varying parameter count.
21// We would pass a GL function pointer and its parameters, and the function would call the GL function
22// with the given varying number of parameters. However, logging is custom in each function, so at least
23// that couldn't be generalized. User could still provide error string format specifier string to the
24// function from outside but then it would be also difficult to pass with parameters should the logger
25// take is input during parsing the format string.
26// So I stick to redundant copy-paste code here. Only a few GL functions get wrappers like this anyway.
27
